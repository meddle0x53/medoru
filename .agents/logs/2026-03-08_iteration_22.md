## Key Features

### Writing Validation
- Stroke count check (exact match required)
- Shape comparison using point distance algorithm
- Normalized coordinate system for fair comparison
- 70% accuracy threshold to pass

### User Experience
- Smooth drawing with pressure-sensitive strokes
- Grid background helps with proportions
- Undo last stroke (not whole canvas)
- Clear canvas to start over
- Shows correct stroke animation after wrong answer
- "Hide preview" button to try again

### Test Integration
- Writing steps appear after multichoice steps
- Worth 5 points vs 1 point for multichoice
- Wrong writing steps go to end of queue (adaptive retry)
- Completes lesson when all steps answered correctly

## Technical Details

### Stroke Data Format
```javascript
// User strokes: array of point arrays
[[{x: 100, y: 50}, {x: 102, y: 52}, ...], ...]
```

### Validation Flow
1. User draws kanji on canvas
2. JavaScript captures strokes as point arrays
3. Strokes sent to server via `submit_writing` event
4. `WritingValidator` compares against KanjiVG data
5. Result returned: `{:ok, accuracy}` or `{:error, reason}`
6. If wrong, stroke preview shown with "Try Again" option

### UI States
- Normal: Show canvas, ready for drawing
- Submitting: Disable controls while validating
- Correct: Green feedback, auto-advance
- Incorrect: Red feedback, show stroke preview option
- Preview: Animated stroke playback, "Hide" to retry
