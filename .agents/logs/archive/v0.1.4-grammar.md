# Iteration: Grammar Test Taking for Students

**Status**: ✅ COMPLETE
**Completed**: 2026-03-31
**Version**: v0.1.4

## Overview

Students can now view and take tests containing grammar steps. Tests can mix vocabulary and grammar steps.

## Completed Features

### 1. Sentence Validation (Pattern Builder)
- **Points**: 2 base per pattern (10 for full lesson)
- **Attempts**: Up to 4 attempts with penalty system
- **Scoring**: Degrades with hints and wrong attempts
- **Mechanism**: Student enters sentences, validated against pattern
- **Alternative Forms**: Supports contracted forms (e.g., 来なく→来な for [Verb-nai] + くて)

### 2. Text Input Questions
- **Points**: 2
- **Mechanism**: Student types answer with hints available

### 3. Multichoice Questions
- **Points**: 1
- **Mechanism**: Student selects from 4-8 options

### Alternative Forms (Key Innovation)
Added `alternative_forms` array to `word_conjugations` to handle Japanese contracted forms:

**Example**: Pattern `[Verb-nai] + くてもいいです` expects `来なく` but students type `来な` (from `来ない`)
- Full form: 来ない (nai-form)
- Contracted form: 来な (used when combining with くて, ければ, etc.)

**Implementation**:
1. Migration adds `alternative_forms` column
2. `Maintenance.Conjugations.generate_alternative_forms/2` creates alternatives
3. `ValidatorCache` preloads alternatives as separate lookup keys
4. `Grammar.Validator` checks both main and alternative forms
