# Medoru Release Guide

## Building a Release

```bash
# Create a production release
MIX_ENV=prod mix release

# Or with explicit version
MIX_ENV=prod mix release --version 0.1.1
```

## Release Structure

After running `mix release`, a `_build/prod/rel/medoru/` directory is created:

```
_build/prod/rel/medoru/
├── bin/
│   ├── medoru          # Release start script (generated by mix release)
│   ├── medoru.bat      # Windows version
│   └── medoru_cli      # Our custom CLI wrapper
├── erts-14.2.5/        # Erlang runtime
├── lib/                # Compiled BEAM files
├── releases/
│   ├── 0.1.0/
│   └── start_erl.data
└── tmp/
```

## Using the CLI in Production

### Option 1: Use `bin/medoru eval` (Standard Phoenix)

```bash
cd _build/prod/rel/medoru

# Make admin
bin/medoru eval "Medoru.Release.make_admin(\"user@example.com\")"

# Run migrations
bin/medoru eval "Medoru.Release.migrate()"

# Show version
bin/medoru eval "Medoru.Release.version()"
```

### Option 2: Use our custom wrapper (Recommended)

For a cleaner experience, copy our CLI wrapper to the release:

```bash
# After building release, copy the wrapper
cp bin/medoru _build/prod/rel/medoru/bin/medoru_cli

# Then use it
cd _build/prod/rel/medoru
./bin/medoru_cli make_admin user@example.com
./bin/medoru_cli version
./bin/medoru_cli migrate
./bin/medoru_cli start
```

### Option 3: Docker

```dockerfile
# Dockerfile
FROM hexpm/elixir:1.19.5-erlang-27.2.5-alpine-3.21.2 AS builder
# ... build steps ...

FROM alpine:3.21.2
WORKDIR /app
COPY --from=builder /app/_build/prod/rel/medoru .
COPY bin/medoru /app/bin/medoru_cli

# Entry point
ENTRYPOINT ["/app/bin/medoru"]
```

Then:
```bash
docker run medoru make_admin user@example.com
docker run medoru start
```

## Production Deployment

### 1. Build Release

```bash
# On build machine
export MIX_ENV=prod
export SECRET_KEY_BASE=$(mix phx.gen.secret)
export DATABASE_URL="postgres://user:pass@localhost/medoru_prod"

mix deps.get --only prod
mix compile
mix assets.deploy
mix release
```

### 2. Deploy

```bash
# Copy to server
scp -r _build/prod/rel/medoru user@server:/opt/medoru

# SSH to server and setup
ssh user@server
cd /opt/medoru

# Run migrations
bin/medoru eval "Medoru.Release.migrate()"

# Seed database (first time only)
bin/medoru eval "Medoru.Release.seed()"

# Create first admin
bin/medoru eval "Medoru.Release.make_admin(\"admin@example.com\")"

# Start application
bin/medoru daemon

# Or with systemd
systemctl start medoru
```

### 3. Managing the Running App

```bash
# Check status
bin/medoru pid

# Remote console
bin/medoru remote

# Stop
bin/medoru stop

# Restart
bin/medoru restart
```

## Environment Variables

Required in production:

```bash
# Database
export DATABASE_URL="postgres://user:pass@host:5432/medoru_prod"

# Phoenix
export SECRET_KEY_BASE="your-secret-key-base"
export PHX_HOST="medoru.example.com"
export PORT=4000

# OAuth (if using)
export GOOGLE_CLIENT_ID="..."
export GOOGLE_CLIENT_SECRET="..."

# File uploads (avatars, etc.)
export UPLOADS_DIR="/var/opt/medoru/uploads"
```

### Uploads Directory

The `UPLOADS_DIR` environment variable specifies where uploaded files (like user avatars) are stored. 

**Important:** This directory must:
- Exist and be writable by the application user
- Be outside the release directory (releases are read-only)
- Be persisted across deployments

**Example setup:**
```bash
# Create uploads directory
sudo mkdir -p /var/opt/medoru/uploads
sudo chown -R medoru:medoru /var/opt/medoru/uploads
chmod 755 /var/opt/medoru/uploads
```

If not set, the default is `/var/opt/medoru/uploads`.

## Systemd Service

```ini
# /etc/systemd/system/medoru.service
[Unit]
Description=Medoru Japanese Learning Platform
After=network.target postgresql.service

[Service]
Type=simple
User=medoru
Group=medoru
WorkingDirectory=/opt/medoru
Environment="MIX_ENV=prod"
Environment="DATABASE_URL=postgres://medoru:secret@localhost/medoru_prod"
Environment="SECRET_KEY_BASE=your-secret"
Environment="PHX_HOST=medoru.example.com"
Environment="PORT=4000"
ExecStart=/opt/medoru/bin/medoru start
ExecStop=/opt/medoru/bin/medoru stop
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Enable and start:
```bash
sudo systemctl enable medoru
sudo systemctl start medoru
sudo systemctl status medoru
```

## Database Migrations on Deploy

```bash
# Migrate before starting new version
bin/medoru eval "Medoru.Release.migrate()"

# Or with our wrapper
./bin/medoru_cli migrate
```

## Creating Admin Users in Production

```bash
# Via eval
bin/medoru eval "Medoru.Release.make_admin(\"admin@example.com\")"

# Via wrapper
./bin/medoru_cli make_admin admin@example.com
```

## Troubleshooting

### Release won't start

```bash
# Check logs
bin/medoru start

# Or run in foreground to see errors
bin/medoru start_iex
```

### Database connection issues

```bash
# Test connection
bin/medoru eval "IO.inspect(Medoru.Repo.query!(\"SELECT 1\"))"
```

### Erlang cookie issues

```bash
# Generate new cookie
bin/medoru eval ":erlang.set_cookie(:erlang.node(), :crypto.strong_rand_bytes(32))"
```
